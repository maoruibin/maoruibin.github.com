![](https://gudong.s3.bitiful.net/img/2e216fed.png)

前段时间我写过一篇文章，讲怎么用 AI 清理磁盘垃圾，那次清了 [72 个 G](https://mp.weixin.qq.com/s/4aTqhjvNKevmRdiR7KQFAw)。

写完我以为能消停一阵。

结果没过多久，系统又弹窗提示存储空间不足了。

微信也跟着提示，说缓存太多需要清理。

这次我直接打开 Codex，在对话框里敲了一句话：

帮我全面扫描磁盘，把所有长期不用的垃圾文件都清理出来。

然后就去倒了杯水。

回来一看，它已经扫描完了，给了我一份详细的汇报：哪些文件夹占了多少空间，哪些是很久没动过的，哪些是缓存可以清。

![图 2：Codex 扫描出的磁盘占用汇报](https://gudong.s3.bitiful.net/img/5b4eb7f2.jpg)

我扫了一眼，大体没问题，让他不要把《天道》删了，模拟器什么的，很久不用了，都删除了。

![](https://gudong.s3.bitiful.net/img/30b34521.jpg)

![图 3：清理完成](https://gudong.s3.bitiful.net/img/8135da6c.jpg)

最终清出来多少？

**94G。**

比上次还多了将近 20G。

### 1. 这次和上次有什么不一样

上次清理的时候，我让 AI 写了一个网页，搭了一个本地服务，通过浏览器去扫描和清理。

能用，但绕了一圈。

这次直接在 Codex 里干。它本身就能访问我的文件系统，省去了所有中间环节。

而且这次它找到了很多上次没发现的东西：

- 3 个 iOS 模拟器，加起来几十个 G，我已经很久没用了
- 2 个安卓模拟器，同样是吃灰状态
- 闪电说这个 App 下面占了 9G 的缓存空间，但我现在基本不用了

这些东西，上次都没清到。

另外还有个细节。缓存文件有个特点——你清了它还会再生成。

所以这次我跟 Codex 补了一句：清理缓存的时候，看下时间，只清老的，别把最近的也删了。

它照做了。

这就是 Agent 和传统工具的一个本质区别：**你可以随时补充条件，它听得懂。**

而且它不是一锅端地把所有东西列给你，而是分好类：有些直接建议清理，有些标成"需要确认"，还有些会提醒你"清了又会回来"。

既智能，又强大，还知道分寸。

这个功能不是天天用，但偶尔来一次，清出几十个 G 的空间，真的很爽。

### 2. 传统清理软件为什么这么弱

我电脑上其实装了腾讯的清理软件。

它每次提醒我，能清多少呢？

1G。有时候 2G。

94G 和 1G，差了将近一百倍。

为什么差距这么大？

传统清理软件的工作原理是这样的：它后台有一份规则文件，里面写好了"这个路径是缓存，可以清"、"那个后缀是临时文件，可以删"。规则从云端同步下来。

问题就在这。**规则永远是滞后的。**

一个新的 App 装上来，它的缓存目录叫什么、数据存在哪，规则文件还没更新呢。它看到一个不认识的文件夹，只会跳过。

所以那些模拟器、闪电说的 9G 缓存、各种奇奇怪怪的本地数据，它全都不认识。

当然，传统软件也有一个优点：**它是可预测的。** 它只删规则允许的东西，不会乱来。

代价就是——太保守了，宁可漏掉 90G 不清，也不敢多动一个文件。

### 3. Agent 为什么能做得更好

Codex 清理磁盘的方式完全不一样。

它拿到一个文件夹路径，不会急着删。它会先看创建时间、最后访问时间、文件类型，然后结合文件夹名字推断这个 App 你还在不在用。

然后它给出一个判断："这个模拟器你已经三个月没启动了，建议清理。"

这不是规则匹配，这是**理解和推理**。

> **一个重要的提醒：清理有风险，删之前一定要自己确认。**
>
> AI 可能会有幻觉，它觉得是垃圾的东西，可能恰好是你需要的。尤其是代码项目、工作文档这类东西，删之前务必看一眼。
>
> 我每次都是让它先扫描汇报，我看一遍确认，然后才让它执行。别因为一篇文章把重要文件给删了，那就不好了。

**人负责把关，Agent 负责干活。**

### 4. 不只是清理软件

清理完之后我就在想：这件事的本质是什么？

磁盘清理，说白了就是"整理"——把有用的留下，没用的清掉。而整理和分析，恰好是 AI 最擅长的事情。

文档整理、数据分析、报告生成、代码重构……这些任务有一个共同特征：**不需要每天干，但干一次价值很高。**

一个月清一次磁盘，一次清出 94G。一个季度做一次数据分析，一次理清整个季度的趋势。

反过来想，那些"高频低门槛"的事——吃完饭记一笔账、随手拍张照、发条消息——这些 Agent 搞不定。不是技术不行，是交互链路太长了。

### 5. 一个光谱

所以我觉得，Agent 和传统软件之间，不是"谁替代谁"的问题。

![图 4：Agent 与传统软件的光谱](https://gudong.s3.bitiful.net/img/e3f80bd2.png)

**更像是一个光谱。**

一端是会被 Agent 完全吃掉的：磁盘清理这种低频工具，甚至包括 IDE、PPT、Word。

中间是混合态：工具负责收集，Agent 负责分析。比如记账 App 负责每天三秒记一笔，月底让 Agent 帮你做消费分析。

另一端是工具不可替代的：需要实时响应、强隐私保护、深度个性化的场景。聊天、谈判、即时通讯——这些你不会想让 Agent 代劳。

判断标准其实很简单：**高频低门槛的事，Agent 干不了；低频高价值的事，Agent 无敌。**

### 6. 产品思维 vs 服务思维

传统软件是**产品思维**。预判用户需求，固化成功能，靠规模化摊薄成本。一个清理功能做出来，服务一百万用户。

Agent 是**服务思维**。你说需求，它实时响应，每次都可以不一样，天然个性化。

但传统软件有一个 Agent 目前还打不穿的东西：**入口和习惯。**

你知道聊天就打开微信，想打电话就点电话图标。它装在那里，一个图标就把交互成本压到最低。而用 Codex，你得先打开它，输入一段话，等它响应——这个链路长太多了。

入口和习惯，才是传统软件真正的护城河。

对我自己来说，结论很简单：**再也不会去下载桌面清理软件了。**

不管是 Mac 还是 Windows，打开 Codex，说一句话，比任何清理软件都好用。
