昨天聊到软件开发，我接着讲。

现在用 AI 开发软件的门槛确实越来越低，催生了很多极度个人化的小工具。

### 72G 磁盘解封

今天早上电脑直接提示空间不足。我打开磁盘工具看了一眼，Mac 自带的只能找出一些大文件，扫得不够彻底。以前也这样清理过，删完没两天，垃圾又卷土重来。

为什么 CleanMyMac 之类的通用工具扫不出来？因为它们不懂"开发者"的垃圾——.pub-cache、DerivedData、未完成的 Chrome 临时下载残余，这些只有开发者自己才知道在哪。

于是我想，干脆让 AI 帮我写个清理工具。

### 让 AI 来做

我告诉它需求：扫描整个磁盘，挖出那些隐蔽的大文件。还把磁盘工具提示的大文件列表截图发过去，让它明确找什么样的目标。技术方案、UI 设计全让它自己发挥。

它选了一套 React 前端 + Node.js 本地守护进程的方案。Node.js 负责深入系统底层扫描和执行物理删除，React 负责呈现界面。普通的 HTML 页面受浏览器沙盒限制，是没办法读取和删除本地文件的，所以必须走本地服务的方案。



我又把 DeepSeek 的 API Key 给了它，让它加入 AI 智能分析。点击之后，DeepSeek 会自动分析每个目录的作用，智能判断哪些可以安全清理，不用我自己一个个去猜。

没一会儿，页面就出来了。说实话还挺好看，功能也很强。

最关键的是，它真把我硬盘里的垃圾翻了个底朝天。最终扫描出来的总垃圾量高达 **72.04 GB**，其中仅 Gradle 缓存就占了 **46.17 GB**——全是编译缓存，删得干干净净。



### 双击就能用的桌面 App

后来还做了一件更酷的事——利用 macOS Bundle 规范和 Chrome App 模式，把它封装成了一个双击即可运行的独立桌面客户端 CleanTrashAI.app，还能拖进 Dock 栏。

不需要懂 Swift 编程，不需要支付苹果 99 美元/年的开发者证书，却拥有了原生软件的即视感。



### 极度个人化

这个例子特别好。工具不需要发布，我可能就用几次，后面边用边改，纯粹为自己服务。

我提供上下文——截图、垃圾路径，AI 负责搬砖，DeepSeek 负责给文件做安全体检。就像一个合伙人。

其实每个人身边都有这种场景，有时候就是一句话的事儿。把诉求告诉 AI，给足恰当的上下文，它就能帮我们把这样的软件鼓捣出来。

我们正在从软件的消费者，变成创造者。
