![封面](https://gudong.s3.bitiful.net/weimd/cover.png)

今天下午我在小龙虾里做了一件事，觉得挺有意思。

inBox 笔记有一个功能叫 API，这个功能上线有一段时间了，但说实话，用的人不多。不是功能不好，是门槛太高——你得会写代码。

什么是 API？说直白一点，就是一个网址。你往这个网址发一段文字，它就帮你存到 inBox 笔记里了。就这么简单。

每个 PRO 用户都有这么一个专属网址。理论上，你在任何设备上，任何场景下，只要能访问网络，调一下这个地址，笔记就存上去了。

听起来很好，但问题来了——普通人看到一串网址，不知道怎么用。

以前我也意识到这个问题，所以专门做了一个网页插件，大家在网页上写笔记，点一下保存，插件会自动把内容通过那个网址传上去。但这还是有点麻烦，你得打开那个网页才行。

直到最近，各种 AI 客户端越来越多。Claude Code、小龙虾、WorkBody、字节的、百度的，桌面端的智能体工具一个接一个。它们都有一个共同的特点——支持技能，而且可以自动帮你调 API。

我今天下午突然想，inBox 笔记的 API 不正好能用上吗？

于是我就试了一下。把 API 的文档直接发给小龙虾，跟它说，你看一下这个文档，是干啥的。它看完就明白了。然后我说，帮我写个技能，以后我说"保存到 inBox 笔记"，你就帮我把内容存上去。

它就真的帮我写好了。技能里把 HTTP 请求封装好了，API 地址也配进去了。

以后不管我在干嘛，只要跟小龙虾说一句"保存到 inBox 笔记"，它就帮我存了。

不需要写代码，不需要懂什么 HTTP，甚至不需要知道 API 是什么。会说话就行。

其实任何一个 AI 客户端都行，WorkBody、小龙虾、Cursor，都支持技能，操作方式差不多——把 API 文档扔给 AI，跟它说你要干嘛，它帮你搞定。

说到这，我觉得这件事的意义不只是 inBox 笔记这一个 API。

以前 API 这个东西，就是程序员的专属工具。普通人碰不到，也不需要碰。但现在有了 AI，AI 充当了中间那个翻译，你说人话，它帮你翻译成 API 请求去执行。

这个门槛一下就没了。

inBox 笔记的 API 一直在那，从做出来的那天起，就是希望用户能在任何场景下记录。浏览器里、Android Studio 里、写代码的时候、甚至开发插件的时候，只要能访问网络，都能把笔记存进来。但以前只有程序员能用。

现在不一样了。

所以如果你是 inBox 笔记的 PRO 用户，平时也在用 AI 工具，可以试试。把 API 文档发给你的 AI，让它帮你搭好技能，以后想到什么，说一句，就存好了。

挺好玩的，以上，共勉。
