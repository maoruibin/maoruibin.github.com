### Vibe Coding 时代的新问题

不知道你有没有这种感觉：

自从用上 Claude Code，开发效率起飞：

- 早上和 AI 讨论架构，10 分钟搞定方案
- 中午 AI 帮我修 Bug，秒级定位问题
- 下午和 AI结对编程，一天写出过去三天的代码

但是...

**周五写周报时，脑子里一片空白："我这周都干了什么？"**

翻 Git 记录？全是 AI 写的代码，看不出自己干了啥
翻聊天记录？和 AI 的对话满天飞，找不到重点
翻 Jira 禅道？只记录了 Ticket，没有过程

### 问题出在哪？

Vibe Coding 时代，我们的工作方式变了：

| 传统开发 | Vibe Coding |
|---------|-------------|
| 写代码 → 提交 | 和 AI 讨论方案 |
| 自己查文档 | AI 帮你调试 |
| 手写测试 | AI 生成代码 |
| 工作有痕迹 | **工作隐于对话** |

**"对话即工作"**——和 AI 讨论的方案、调试的过程、解决的问题...这些都是有价值的工作，但留不下痕迹。

### 我的解决方案：devlog 技能

我给 Claude Code 写了一个技能 **devlog**，让 AI 帮你记录工作，无需打断流程：

```
你: 记一下，刚才修复的 Feed 缓存问题
Claude: ✅ 已归档到 ~/devlog/2025-01-21.md
```

**就这么简单**——继续和 AI 协作，它会在合适的时机帮你记录。

### devlog 是什么？

devlog 是一个 Claude Code 技能，核心能力：

1. **对话式记录**——自然语言触发，AI 自动理解分类
2. **结构化存储**——按 incident/feat/design/ops/bug/learn 分类
3. **灵活存储**——全局流水账 + 项目本地文档
4. **一键周报**——自动汇总本周工作

### 快速上手

```bash
# 安装
git clone https://github.com/maoruibin/devlog.git ~/.claude/skills/devlog

# 配置 SKILL.md（见仓库说明）
```

然后在 Claude Code 中直接对话：

```
你: 记一下，刚才和 AI 讨论的用户中心重构方案，存到项目里
Claude: ✅ 已归档到 ./.devlog/2025-01-21.md

你: 记录刚才修复的登录 token 过期问题
Claude: ✅ 已归档到 ~/devlog/2025-01-21.md
```

### 工作流示例

```bash
# 早上：和 AI 讨论架构
你: 帮我设计用户中心的重构方案
AI: [分析后给出方案]
你: 记一下这个方案，存项目里
Claude: ✅ 已归档到 ./.devlog/2025-01-21.md

# 下午：AI 帮忙修 Bug
你: 登录 token 总是过期，帮我查查
AI: [定位问题]
你: 记一下这个 Bug
Claude: ✅ 已归档到 ~/devlog/2025-01-21.md

# 周五：生成周报
devlog weekly
```

### 周报长什么样？

```
📊 周 报 / Weekly Report
==================================================

📐 **技术方案** (2)
  • 用户中心重构方案
  • Feed 缓存策略设计

🐛 **常规Bug** (3)
  • 修复登录 token 过期
  • 修复点赞数不刷新
  • 修复首页 Crash

🚨 **线上故障** (1)
  • 支付服务超时
    Root Cause: 数据库连接池耗尽
    Fix: 扩容 + 慢查询优化

────────────────────────────────────────────────────
📅 2025-01-15 ~ 2025-01-21  ·  共 5 天有记录
```

### 设计理念

**1. 对话即触发**

不需要切换工具，不需要打断思路，自然对话即可记录：

```
"记一下..."
"存档..."
"log this..."
"记录到日志..."
```

**2. AI 自动分类**

你只需要描述内容，AI 自动判断分类：

```
"首页 Crash 了" → incident
"加个点赞功能" → feat
"怎么设计缓存" → design
```

**3. 结构化但不僵硬**

支持 6 种分类，覆盖开发者常见工作场景，同时支持自定义 detail 记录细节。

### Vibe Coding 时代的开发日志

传统开发日志只记录代码变更（Git commit），但在 AI 辅助开发的今天：

- **对话即设计**——和 AI 讨论的方案是宝贵的设计文档
- **调试即学习**——AI 帮你排查问题的过程值得记录
- **快速迭代**——一天完成的工作更多，更需要记录

devlog 让这些"隐性工作"显性化，让你的每一天都清晰可见。
