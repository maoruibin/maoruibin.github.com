最近几个月，我的编程工具发生了很大的变化。

之前一直在用 Claude Code，是一个终端工具。我也在公众号里写过好几次，运行效率确实高。但这几个月，编码智能体的工具发展太快了，我忍不住想聊聊自己的感受。

### 从命令行到桌面

五月份开始，我先下载了 OpenAI 的 Codex，这是一个桌面的 AI 编码智能体。后来赶上 Google I/O 大会，又关注到了 Google 的 Antigravity 2.0，也是桌面编码智能体。

这之后我买了 Google AI Pro，从此打开了新世界。

用了一段时间之后，有一个很强烈的感受：桌面 Agent 跟命令行工具比，体验提升太大了。

在终端那个黑框框里，命令行执行效率高，但操作效率其实很低。你只能看文字，没法直观地看到结果。而在桌面 APP 里就不一样了，我们已经习惯了通过微信、QQ、Word 或者网页去操作，熟悉的 GUI 界面，写需求很轻松，也能直观地看到结果。

这个差距，用 Claude Code 的时候感受特别明显。

### 计划文件和批注

再举一个例子。Project IDX 在处理稍微复杂的需求时，默认会先生成一个计划文件，让你 review，没问题就点执行。

我觉得这个特别好。

以前在 Claude Code 里也有类似的功能，但很难查看和修改。而在 Project IDX 里很容易修改，左右侧边栏可以预览计划文件，还能对计划文件进行批注。

批注功能特别特别好。

### Codex 和 DeepSeek

后来我也用了 Codex，但有一个不太方便的地方——它需要 ChatGPT 账号。我目前还没有，就用 DeepSeek 的 API 做替代，体验也不错。

不过因为 Project IDX 有 Google AI Pro 的套餐，额度多到几乎用不完，所以我还是主要在用 Project IDX。Codex 只偶尔用来做做 PPT 或者研究尝试。

如果哪天谷歌账号出问题，我可能还是会切回 Codex。

### 速度和效率

总之，最近我主要用 Google 的 Project IDX，开发效率特别高。

Google I/O 上发布的 Gemini 3.5 Flash 版本，编码速度非常快，比我之前用 GLM 快不少。最近写了好几个 APP，都是让它帮我写的，体验很好。

### 不只是写代码

这种可视化的桌面 Agent 变得越来越好，也越来越容易被大家接受。它不仅仅可以用来写代码，做 PPT、操作网页其实都可以。

我现在只用了一部分功能，还有很大的探索空间。

工具的变化很快，AI 的变化也很快，总之是变得越来越好用。从用户角度，什么好用我就用什么。

我是咕咚，愿工具让你更自由。
