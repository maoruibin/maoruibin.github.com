# 小龙虾的热潮在退去，但我还在用

两个月前，小龙虾（OpenClaw）横空出世的时候，那场面确实有点夸张。

国内厂商纷纷跟进，腾讯、字节都在搞，火到各个大厂都要蹭一波流量，甚至有人在闲鱼上挂 500 块教别人安装。

但最近一个月，明显感觉热度在降温。

其实原因也不难理解。上手成本高，很多人折腾半天装上了，结果不知道拿来干嘛。记忆机制是个硬伤，聊着聊着就忘了之前说过啥。再加上 Hermes Agent 出来分流了一批用户——那是 Nous Research 做的自学习 AI 智能体，主打一个自己摸索自己学，概念挺新的，确实吸引了不少人。

不过热度退了不代表没人用了，我自己就还在用，而且用得挺舒服。

主要两个场景。

第一个是小麻雀，我的写作助手。每天有灵感或者想法的时候，直接录一段语音发过去，小麻雀帮我把录音转成文字，润色一下，生成封面图，然后发到公众号和博客。说实话，这个流程跑顺之后，写东西的阻力小了很多，想到什么就能马上记下来，不用担心"写不好"。

第二个是小诸葛，研究分析助手。看到一篇好文章，把链接丢给小诸葛，它帮我把文章读完，做个总结，再写一份深入浅出的分析报告。对于我这种每天要看不少技术文章的人来说，省了不少时间。

至于小码牛，也就是小龙虾里的代码智能体，我用了一段时间，最后还是回到了 Claude Code。倒不是说小码牛不好用，而是在小龙虾里写代码，效率确实比直接开 Claude Code 低一点。写代码这件事，还是 Claude Code 更顺手。

不过话说回来，不管是小龙虾、Claude Code 还是 Hermes Agent，最关键的其实就一点：用起来。

工具没有好坏之分，它们一直都在迭代。今天觉得这个好，明天可能那个又升级了。与其纠结选哪个，不如先挑一个开始用，用着用着就知道自己需要什么了。

说直白一点，大语言模型是大脑，但光有大脑不够，得有手脚去干活。小龙虾和 Claude Code 这类智能体，就是让 AI 长出手脚的东西。

我觉得这是未来的趋势，不是可能，是确定的方向。
