![封面：Codex 用上智谱 GLM 了，升级 CC Switch 就行](https://s3.bitiful.net/gudong/images/codex-glm/cover.png)

前天花了一两个小时，在微信、GitHub、推特上翻了个遍，就想搞清楚一件事——怎么在 Codex 里用智谱的 GLM 模型。

我自己买了智谱的 Coding Plan，模型能力没得说，但就是没找到一篇说清楚的教程。

最后放弃了。

结果今天早晨，CC Switch 更新了。看了眼 Release Notes——3.16.2 版本对 GLM 做了适配。

之前 3.16.1 不行，必须升到 3.16.2。

升级之后怎么配？跟之前配 DeepSeek 一模一样。打开 CC Switch，选智谱，填上 API Key，搞定。

![图 1：CC Switch 配置智谱 GLM 成功](https://s3.bitiful.net/gudong/images/codex-glm/01-zhipu-ccswitch-success.png)


不用折腾代理，不用研究转发工具。升个级，填个 Key，直接用。

有一点需要注意：智谱 GLM 的响应速度比 DeepSeek 稍慢一些，体感上能感觉到差别。但能用，而且稳定。

---

如果你也在用智谱的 Coding Plan，一直想在 Codex 里用起来——现在可以了。

CC Switch 升到 3.16.2，配置跟 DeepSeek 一样，选 GLM 就行。

就这么简单。

还有一点，也是我踩过的坑——不要在 Codex 里让它处理图片。

不管是 DeepSeek 还是 GLM，只要对话里出现图片，Codex 就容易卡死。尤其是 DeepSeek，本身就不支持多模态，你给它发图片，它直接就懵了。GLM 我也不太确定具体原因，但实测下来也一样，一发图片就容易挂。

所以现在的经验就是：用 Codex 写代码的时候，就老老实实写代码，别往对话里塞图片。
