其实之前我已经写过好几篇文章，给大家安利 Codex 了。

![之前写过的 Codex 相关文章截图](https://gudong.s3.bitiful.net/img/cdcd58d7.png)

Codex 确实好用，我自己每天在用，也折腾了不少——从 DeepSeek 到智谱 GLM，一路配下来，确实觉得它是个好东西。所以我逢人就推荐。

但说实话，用 Codex 确实有个麻烦的点。

下载完软件，你还得再装一个 CC Switch，去配置模型、填 API key、改本地路由。

这些我自己折腾过，也写成了教程，可每次有人问我“怎么用”，我都得发一堆链接过去，对方看完经常就放弃了。

如果没有稳定的 OpenAI 账号、网络和付费条件，你很难稳定用上原生的 GPT-5.5。

> 如果你这些都已经配好了，那这篇文章不适合你，你继续用 Codex 就行。

我写这篇，是给另一拨人看的。

### 卡在“用起来”的那拨人

很多是老师、体制内的同事、做传统行业的同学。他们看完文章觉得这东西好，也想试，可一打开教程就懵了——CC Switch 是什么？API key 去哪注册？本地路由又是什么？

愿意折腾的人，折腾通了确实有用。

但多数人卡在“用起来”这一步，就再也没往下走。

我一直想，国内怎么就没人做一个更适合普通人上手的 Codex 出来？

好东西已经在那里了，真正缺的是把它做成下载就能用。

### 评论区送来的答案

昨天我那篇文章下面，有个哥们评论说：试试 ZCode 呢，做得还不错。

![读者评论推荐 ZCode 的截图](https://gudong.s3.bitiful.net/img/15b9269b.png)

ZCode 是智谱出的一个桌面智能体。我其实之前下载过，当时觉得一般，就没再用。

今天又下载下来试了一下。一打开界面，我乐了。

这不就是 Codex 吗？

![ZCode 主界面截图](https://gudong.s3.bitiful.net/img/7d4c644f.png)

界面看得出来明显借鉴了 Codex，布局、交互、操作逻辑都很接近。对于一个被 Codex 折磨过配置的人来说，打开就能认出来，几乎没有学习成本。

### 很像，但还没全

当然，它现在还没把 Codex 的功能全搬过来。

比如批注、定时任务，这些 Codex 里很好用的能力，ZCode 还没有。

我猜他们接下来大概率会继续补这些能力。

Codex 已经把桌面 Agent 的核心交互跑通了，ZCode 现在沿着这条路补齐，用户是受益的。

作为用户，我觉得这事挺好。

### 真正让我舒服的一点

界面像不像 Codex，其实是次要的。真正让我觉得舒服的，是它把“用起来”这件事的门槛，直接砍掉了。

你买智谱的 Coding Plan，不管是 Lite 还是 PRO，装好软件、登录账号，就能用。

不用 CC Switch，不用配路由，不用满世界找 API key。

我自己一直买的智谱套餐，所以直接用 GLM-5.2，开箱即用。

如果你觉得智谱的套餐贵，它也支持换别的模型。理论上去 DeepSeek 注册一个 key，或者用小米的模型，都能接上。

![ZCode 模型配置界面截图](https://gudong.s3.bitiful.net/img/09254a54.png)

核心工作流跟 Codex 已经很接近，普通人上手基本够用了。

### 我对它还有一点期待

当然，期待还是有的。

我最希望它支持类似 Codex 的侧边聊天。

![Codex 侧边聊天功能截图](https://gudong.s3.bitiful.net/img/420f12a5.png)

现在很多时候，我会想同时开两个窗口聊。一个窗口继续跑主线任务，另一个窗口拿来追问细节、整理材料，互不打断。

第二个是对话分叉。

![Codex 对话分叉功能截图](https://gudong.s3.bitiful.net/img/37e2d06e.png)

一个会话聊到中间，经常会冒出好几个方向。最好能从当前会话直接分叉出另一个会话，带着已有上下文继续探索。这样就不用在一个长会话里来回拧，最后把上下文搅成一团。

第三个是批注和引用。

有时候 AI 生成了一段信息，我只想选中其中一句做批注，或者直接引用到下一轮提示词里。这个能力看起来不大，但对提示词精准度很有帮助。

我之前特意写过一篇文章讲：[Codex 的侧边栏设计的真好](https://mp.weixin.qq.com/s/N2vHtTQg-JF1zX4vCE4X_w)

除了这三个，还有几个我也很期待：自动化、定时任务、手机端。

这些都是重度 Codex 用户会反复用到的硬核功能。我估计他们应该也已经在做了。

### 不过有一点我得夸一下

缺归缺，有一处细节我得单独夸一下：它的输入框支持标签，通过 # 来选择一个会话。

![ZCode 的 # 会话关联功能截图](https://gudong.s3.bitiful.net/img/0f2e9db9.png)

有时候一个会话跑着跑着异常了，或者我想换一条线继续聊，就可以新建一个会话，然后把之前那个会话 关联进来。前面聊过什么，它能被更自然地引用过来。

Codex 其实也能做到类似的事，但 ZCode 把它做成了一个明确的产品入口，这就很舒服。

所以前面的期待，就当我小小许愿一下。如果能被 ZCode 团队看到就更好了。

### 我的建议

如果你之前跟着我的教程试过 Codex，但被 CC Switch 和模型配置劝退了，那我特别推荐你试试 ZCode。

几乎零门槛，下载软件、登录账号，就能开始用。

这就是我一直想要的、国内版的 Codex。

国外版好用是好用，但门槛对普通人太高了。现在终于有个国内版，能把这件事变简单。
