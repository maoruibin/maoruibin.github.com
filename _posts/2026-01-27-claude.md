| 问题 | Claude 官方 | 智谱方案 |

今天这篇文章，教你用**智谱 GLM Coding Plan** 接入 Claude Code，完整走一遍从安装到实战的流程。

---

## 1. 什么是 Claude Code？

在 AI 编程领域，技术发展经历了三个阶段：

1. **代码补全**（如 GitHub Copilot）—— 你写代码，它补全
2. **对话辅助**（如 ChatGPT）—— 你问问题，它给答案
3. **智能代理**（Claude Code）—— 它能自主规划、调用工具、完成复杂任务

**Claude Code 不是"工具"，是"智能体"**。

它不只是写代码，还能：
- 操作文件、运行命令
- 理解整个项目结构
- 跨项目读取信息
- 记住上下文和历史交互

![](https://gudong.s3.bitiful.net/weimd/1769513869118_image.png)

## 1.5 适合谁？不适合谁？

**说在前面：Claude Code 有上手门槛。**

它需要你：
- 会用命令行（终端）
- 愿意折腾环境配置
- 能接受偶尔的报错和调试

**如果你是编程小白**，只是想用 AI 帮忙写代码，我更推荐 **Trae**。

Trae 是国产的 AI 编程工具，有图形界面，开箱即用，对新手友好得多。等你在 Trae 上手了、觉得不够用了，再回来折腾 Claude Code 也不迟。

但如果你愿意跨过这个门槛——**Claude Code 会打开一个新世界**。

它的能力上限远超传统工具，多窗口、Skills、MCP 这些特性一旦用上，就回不去了。

**请妥善保管**，不要泄露给他人。

普通用户，使用 Lite 就够用。入门用这个套餐就好。

## 3. 一键安装：Coding Tool Helper

### 3.1 先安装 Claude Code

**第一步：打开终端**

在你的电脑上搜索「终端」，打开它。

不用害怕，你就把它理解成一个界面很酷的 AI 对话框。

![](https://gudong.s3.bitiful.net/weimd/1769514326178_image.png)


**第二步：运行安装命令**

在终端里粘贴下面的命令，然后回车：

```bash
npm install -g @anthropic-ai/claude-code
```

**如果遇到报错**：
- 如果你已经安装了 Cursor 等 AI 编程工具，强烈建议在这些工具内安装，它们会帮你解决问题
- 如果没有其他工具，可以询问任何 AI 来尝试解决错误

**第三步：备选安装方式**

如果 npm 方式不成功，可以用官方脚本：

**macOS / Linux / WSL：**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows：**
```bash
irm https://claude.ai/install.ps1 | iex
```

**第四步：验证安装**

安装完成后，输入：

```bash
claude --version
```

如果出现版本号，则代表安装成功。


### 3.2 运行 Coding Tool Helper

Coding Tool Helper 是一个编码工具助手，使用它可以快速帮我们将自己的 GLM 编码套餐加载到 ClaudeCode 中，

一键配置脚本：

```bash
npx @z_ai/coding-helper
```

你会看到一个亲切的中文界面，按照提示把 API Key 粘贴进去，一步步完成配置即可。


![](https://cdn.bigmodel.cn/markdown/1764664136830image.png?attname=image.png)

## 4. 第一次使用

### 4.1 启动 Claude Code

创建一个空文件夹，进入后启动：

```bash
mkdir todo-demo
cd todo-demo
claude
```

你会看到欢迎界面，AI 已经准备好听候差遣。

![](https://gudong.s3.bitiful.net/weimd/1769514744263_image.png)

选择 Yes Progress，同意

![](https://gudong.s3.bitiful.net/weimd/1769514763176_image.png)


### 4.2 让 AI 开始干活

直接告诉它你想做什么：

> 帮我开发一个 HTML 的 todo 应用

Claude 会：
1. 分析需求
2. 规划实现方案
3. 开始编写代码
4. 创建文件时**请求你的授权**

![](https://gudong.s3.bitiful.net/weimd/1769514905726_image.png)

### 4.3 授权操作

当 Claude 需要创建文件、运行命令时，会弹出让你确认。

```txt
1. Yes
2. Yes, allow all edits during this session (shift+tab)
3. Type here to tell Claude what to do differently
```
根据自己的情况，选择即可，通过键盘上下键，选择后，回车确认。

后面创建文件，或者执行命令，可能频繁让我们确认，然后打断编码过程，怎么办？

### 4.4 危险模式（可选）

如果你信任这个环境，可以启动时跳过授权：

```bash
claude --dangerously-skip-permissions
```

**新手慎用**。

## 5. 项目初始化：让 AI 更懂你

首次开始新项目时，如果已经有项目代码，可以先执行：

```bash
claude
> init
```

这会扫描整个项目，生成 `claude.md` 文件——相当于给 AI 写了一份"入职手册"。如果还是一个空项目，等到后续再执行 init，或者开始时，告诉你的诉求，执行 /init 【具体要求】 一般不需要。

### 小技巧：让 AI 叫你的名字

在 `claude.md` 中添加：

```markdown
**IMPORTANT: Always address the user by name.**

The user's name is **咕咚** (Gudong). When responding to questions or providing updates:
- Start responses with "咕咚，" or "好的咕咚，"
```

之后 AI 的回复就会变成：

> 咕咚，我已经完成了日志简化的工作...
> 好的咕咚，这个问题是因为...

![](https://gudong.s3.bitiful.net/weimd/1769515227256_image.png)

这样有什么好处：等到某一次你发现回答问题时，AI没有叫咱们的名字，说明他已经把项目上下文丢了，咱们需要重建一次，执行 /clear 命令。

## 6. 常用命令

### 基础命令

| 命令 | 作用 |
|------|------|
| `/exit` | 退出 Claude Code |
| `/clear` | 清空当前对话上下文，重新加载 |
| `/help` | 显示帮助信息 |

### 启动参数

| 参数 | 作用 |
|------|------|
| `claude --compress` | 启动时自动压缩上下文 |
| `claude --dangerously-skip-permissions` | 跳过授权确认 |
| `claude --no-progress` | 不显示进度条 |
| `claude --debug` | 显示调试信息 |

### 上下文管理

当对话很长、上下文太多时，Claude 会自动压缩摘要。

你也可以手动清空：
```
/clear
```

或退出重启：
```
/exit
claude
```

## 7. 多窗口模式：真正的多线程工作

这是我最喜欢的功能。

开两个命令行窗口，**同一个项目**：

- 窗口 A：开发设置页面
- 窗口 B：开发关于页面

它们**互不干扰**，各自干活。

![](https://gudong.s3.bitiful.net/weimd/1769515357761_image.png)

你可以同时开 6 个、8 个窗口，只要跟得上就行。

### Git Worktree 模式

高级用户还可以配合 **Git Worktree** 使用。

Git Worktree 让你同时检出同一个仓库的多个分支到不同目录：

```bash
# 创建一个 worktree 用于开发新功能
git worktree add ../feature-a origin/feature-a

# 在新窗口进入这个目录
cd ../feature-a
claude
```

这样你可以：
- 窗口 A：在主分支修 bug
- 窗口 B：在 feature-a 分支开发新功能

两个窗口完全独立，互不影响。

## 8. 技能（Skills）：把重复工作变成快捷指令

**技能**就是把重复做的事封装成"快捷指令"。

### 8.1 技能示例：解释代码

创建一个解释代码的技能：

```bash
mkdir -p ~/.claude/skills/explain-code
```

编写 `SKILL.md`：

```markdown
---
name: explain-code
description: 用通俗语言和图表解释代码的工作原理
---

解释代码时，请：
1. 用生活中的例子做类比
2. 用 ASCII 图展示流程
3. 逐步讲解代码执行
4. 指出常见误区
```

之后你问"这段代码怎么工作？"，AI 就会用这个技能来回答。

### 8.2 技能示例：代码审查

```bash
mkdir -p ~/.claude/skills/review
```

`SKILL.md`：

```markdown
---
name: review
description: 审查代码，指出潜在问题和改进建议
---

审查代码时，请检查：
1. 命名是否清晰
2. 是否有重复逻辑
3. 错误处理是否完善
4. 性能是否有优化空间
```

### 8.3 真实案例：把 GitHub 项目变成工具

AI 时代真的不一样了。

以前 GitHub 上那些牛项目，对小白来说就是"看得到用不了"。

比如 **yt-dlp**，143k star 的视频下载神器。你知道它强，但看到命令行、环境配置就劝退了。

现在有了 Skills，你可以直接告诉 Claude Code：

> 帮我把 yt-dlp 打包成 Skill

它自己去分析项目、写封装、处理依赖。几分钟后，你就有了一个能用的视频下载工具。


不只是视频下载。格式转化、网页转 APP、密码工具……GitHub 上那些验证过的项目，都能变成一个 Skill。

**这些开源项目的稳定性，远超 AI 临时写的代码。**

### 关于技能的补充

写技能有个原则：**一个技能只做一件事**。

就像乐高积木，每块积木功能简单，但拼在一起可以变化无穷。与其做一个"大而全"的技能，不如做几个"小而美"的技能，用的时候组合起来。

写技能其实不需要你自己写代码。你可以直接让 Claude Code 帮你写——告诉它需求，它就能生成完整的技能代码。

官方甚至提供了一个专门的技能叫 `create-skills`（创造技能的技能），它会一步步问清楚你的需求，然后自动创建技能。

Claude Code 支持多种方式处理图片：

## 10. MCP：插件生态

MCP（Model Context Protocol）是 Claude Code 的插件系统。

一个应用场景：**Figma 设计图转代码**。

安装 Figma MCP 后，把设计稿链接发给 AI，它能直接生成对应的 HTML/CSS 代码。

这里不展开，有兴趣可以搜索 "Claude MCP" 了解更多。

## 11. 推荐工作流：理解新项目

官方文档有个很好的工作流，分享给你：

```bash
# 1. 进入项目目录
cd /path/to/project

# 2. 启动 Claude
claude

# 3. 请求项目概览
> give me an overview of this codebase

# 4. 深入了解
> explain the main architecture patterns used here
> what are the key data models?
> how is authentication handled?
```

这个流程能帮你快速上手陌生项目。

## 12. 遇到问题怎么办？

### 429 错误（用量超限）

```
Usage limit reached for 5 hour. Your limit will reset at 2025-10-18 03:03:23
```

说明套餐用完了，等时间重置，或者升级套餐。我自己之前很快就发现了超额，然后就买了两个账号，也是一种办法，更好的办法是升级套餐。


### 其他问题

[官方中文文档](https://code.claude.com/docs/zh-CN/overview) 写得通俗易懂，遇到问题先查文档。

智谱的 [Coding Plan 文档](https://docs.bigmodel.cn/cn/coding-plan/tool/claude) 也有详细的配置说明。
